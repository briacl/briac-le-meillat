 6    CONFIGURER le NAT (MASQUERADE) sur la Passerelle🔁
    SE CONNECTER SUR : la Passerelle (Ubuntu Server)⚠️ 🖥️
Pourquoi le NAT est nécessaire ?
Situation sans NAT :
  H1 (192.168.0.2) envoie un paquet vers 172.31.25.9 (iut-rt).
  La passerelle le reçoit et le retransmet sur le réseau IUT.
  Le routeur IUT reçoit un paquet SOURCE = 192.168.0.2 ...
  ... mais 192.168.0.2 est une IP PRIVÉE, inconnue sur le réseau IUT !
  Le routeur IUT ne sait pas où renvoyer la réponse => le paquet de 
retour est perdu.
Situation AVEC NAT (MASQUERADE) :
  Quand la Passerelle retransmet le paquet de H1 vers le réseau IUT,
  elle REMPLACE l'adresse source 192.168.0.2 par SA PROPRE IP IUT (ex: 
172.31.x.x).
  Le routeur IUT voit un paquet SOURCE = 172.31.x.x => il sait répondre  
à la Passerelle.
  La Passerelle reçoit la réponse et la retransmet à H1 (192.168.0.2).
  => H1 peut communiquer avec l'extérieur sans que l'extérieur 
connaisse son IP privée.
 
Commande à exécuter sur la Passerelle :
sudo iptables -t nat -A POSTROUTING -o enp0s3 -j MASQUERADE
#   -t nat           : table NAT
#   -A POSTROUTING   : après la décision de routage, avant la sortie
#   -o enp0s3        : paquets sortant par l'interface IUT
#   -j MASQUERADE    : remplace IP source 192.168.0.x par IP IUT 
172.31.x.x
